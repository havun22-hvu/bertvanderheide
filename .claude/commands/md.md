# MD-First Reminder

```
╔═══════════════════════════════════════════════════════════════╗
║  STOP! MD FILES ZIJN LEIDEND                                  ║
║                                                               ║
║  1. LEES eerst de relevante MD files                          ║
║  2. UPDATE de MD files met wat je gaat bouwen                 ║
║  3. WACHT op goedkeuring van de gebruiker                     ║
║  4. PAS DAN code schrijven                                    ║
║                                                               ║
║  Waarom?                                                      ║
║  - Volgende sessie is context weg                             ║
║  - MD files zijn het geheugen tussen sessies                  ║
║  - Zonder docs herhaalt elk probleem zich                     ║
╚═══════════════════════════════════════════════════════════════╝
```

## Wat je NU moet doen

1. **Zoek relevante docs** voor de huidige taak
2. **Meld aan gebruiker** wat je vond (of niet vond)
3. **Update/maak docs** VOORDAT je code schrijft:
   - Wat is de bedoeling?
   - Hoe moet het werken?
   - Hoe ziet het eruit?
4. **Vraag bevestiging**: "Kloppen deze docs?"
5. **Dan pas coderen** vanuit de docs

## Niet doen

- Direct code schrijven
- "Ik fix het even snel"
- Oplossingen verzinnen zonder doc-check
- Docs achteraf bijwerken

## Onthoud

> **Code volgt docs. Nooit andersom.**
> **MD files = geheugen tussen sessies.**
